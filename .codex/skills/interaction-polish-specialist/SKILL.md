# Interaction Polish Specialist Skill

## Purpose

Use this skill to make Pregunton feel smooth, simple and intentional.

Interactions should feel like handling paper cards, not using a dashboard.

## Pregunton interaction principles

- Deck mode should feel like drawing/swiping a card.
- Controls should be minimal.
- Feedback should be quiet.
- Animations should be subtle.
- The user should never wonder what happened.
- The interaction should support reading aloud.

## Required checks

Review:

1. Primary CTA behavior
2. Secondary link behavior
3. Deck navigation
4. Swipe gestures
5. Arrow controls
6. Keyboard navigation
7. Random card behavior
8. Save/favorite behavior
9. Share/copy behavior
10. Toasts
11. Empty states
12. Loading states
13. Reduced motion

## Recommended Pregunton microcopy

Save:
- Carta guardada
- Ya estaba en tus cartas
- Carta quitada

Empty saved cards:
- No tienes cartas guardadas todavía.
- Cuando una pregunta te guste, toca el corazón. Aquí aparecerá.

Random:
- Sacar una carta
- Sacar otra
- Sácame una pregunta

Swipe:
- Desliza para cambiar de carta
- Usa ← → para cambiar

## Motion rules

Use:
- small translate
- tiny rotation
- opacity transition
- card lift on hover
- reduced motion support

Avoid:
- bouncy animations
- excessive spring effects
- large moving backgrounds
- distracting decorative motion

## Output format

### Interaction audit

What works and what feels unclear.

### Microcopy fixes

List exact strings.

### Motion plan

List subtle transitions.

### Accessibility plan

Keyboard, focus and reduced motion.

### Implementation steps

Files/components to update.
