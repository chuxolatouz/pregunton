# Visual Identity Director Skill

## Purpose

Use this skill to define and protect the visual identity of Pregunton before building UI.

Pregunton should not look like a generic SaaS app, dashboard, or default shadcn/ui template.

Pregunton should feel like:
- paper
- cards
- notes
- small decks
- warm ink
- quiet conversation
- tactile objects
- social moments
- a question held in your hand

## Required workflow before coding

Before implementing UI, produce:

1. Visual intent
2. Emotional tone
3. Visual metaphors
4. Color direction
5. Typography direction
6. Surface/material direction
7. Component personality
8. Motion direction
9. Anti-patterns
10. Design token plan

## Pregunton visual rules

Use:
- warm off-white backgrounds
- ink-like dark text
- subtle paper shadows
- card stack depth
- soft borders
- calm spacing
- tactile paper components
- restrained accent colors
- editorial typography
- large readable question cards

Avoid:
- default shadcn look
- generic SaaS gradients
- heavy glassmorphism
- overdecorated scrapbook style
- childish stickers
- dense dashboards
- too many cards above the fold
- visually noisy sections
- excessive icon usage
- meaningless decoration

## Design token expectations

Create or maintain tokens for:

- paper background
- paper elevated surface
- paper border
- ink primary
- ink muted
- accent warm
- accent travel
- accent friends
- accent couples
- accent deep
- paper shadow
- card rotation
- focus ring

## Output format

Before implementation, output:

### Visual Direction

Describe the visual style in 3-5 sentences.

### Token Plan

List colors, shadows, radius, typography and spacing.

### Component Personality

Explain how buttons, cards, tabs, badges and question cards should feel.

### Anti-generic Checklist

List what must be avoided.

### Implementation Plan

List the exact components/files to update.
