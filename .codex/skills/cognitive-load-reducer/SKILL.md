# Cognitive Load Reducer Skill

## Purpose

Use this skill to simplify pages without removing product value.

Pregunton should feel immediate:
one purpose, one card, one action.

## Pregunton rule

Home = silence + purpose + one action
/mazos = discovery and browsing
/mazo/[id] = one question, no distractions
SEO pages = indexable content

## Required analysis

Before editing, identify:

1. Primary purpose of the screen
2. First user action
3. Secondary actions
4. Above-the-fold elements
5. Elements causing cognitive load
6. Elements to remove
7. Elements to move lower
8. Elements to move to another route
9. Elements to hide behind progressive disclosure

## First viewport rule

The home first viewport should contain only:

- minimal header/logo
- one strong headline
- one short paragraph
- one primary CTA
- one subtle secondary link
- one paper/card visual object

Do not show above the fold:

- full deck grid
- all discovery modes
- FAQ
- SEO text blocks
- large lists
- more than two CTAs
- repeated buttons
- dense cards
- mode toggles unless visually tiny and justified

## Progressive disclosure

Introduce complexity in this order:

1. Explain purpose
2. Explain how it works
3. Show moments
4. Show featured decks
5. Link to full library
6. Show SEO/internal links lower on page

## Output format

### Screen purpose

One sentence.

### First action

One sentence.

### Remove / Move / Keep

Three lists.

### New structure

List sections in order.

### Simplified copy

Provide exact copy.

### Implementation plan

List files/components.
