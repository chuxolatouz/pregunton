# SEO Content Architect

## Purpose

Use this skill when creating public pages that need SEO structure, metadata, headings, search intent alignment, and natural content.

## Required SEO Brief

Before writing or implementing a public page, define:

- Primary keyword
- Secondary keywords
- Search intent
- SEO title
- Meta description
- H1
- H2 structure
- Internal link suggestions
- Open Graph title
- Open Graph description
- Canonical URL recommendation
- Schema.org recommendation if applicable
- Natural copy rules
- Multilingual considerations if relevant

Use [seo-patterns.md](seo-patterns.md) for metadata and on-page patterns.

## Project Guidance

- Category pages may target specific search intent.
- Question collections should be indexable where useful.
- Titles should match how people search.
- Balance SEO with a playful, human tone.
- Serve Spanish-first searches first.
- Keep English expansion possible through locale-aware content and routes.
- The interactive question-card experience should enhance the page, not replace all indexable content.
- Add FAQ sections only when useful and visible.
- Structured data must match visible content.
- Avoid keyword stuffing, doorway pages, and repetitive near-duplicate intros.

## Natural Copy Rules

- Write for humans first, search engines second.
- Put the main keyword where it naturally belongs: title, H1, intro, and metadata.
- Use secondary keywords only where they help the page read naturally.
- Keep Spanish accents in visible copy when the project supports them. ASCII slugs may omit accents.
- Avoid repeating the exact keyword in every heading.
- Make the page useful even if the user never opens interactive card mode.
