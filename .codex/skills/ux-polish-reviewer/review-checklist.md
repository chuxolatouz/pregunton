# Review Checklist

## UX

- First action is visible without hunting.
- Secondary actions do not compete with the first action.
- Category/deck purpose is clear.
- Interactive mode has obvious next/copy/share controls.

## UI

- Visual tone matches category.
- Question card is readable and stable on mobile.
- No nested cards or generic SaaS composition.
- No meaningless gradients or decorative clutter.

## Copy

- Spanish feels natural.
- CTAs are concrete.
- No keyword stuffing.
- No forced intimacy or cringe.

## SEO

- Title, description, H1, canonical, and Open Graph metadata exist.
- Question content is visible in HTML where useful.
- Internal links point to related decks.
- Structured data, if present, matches visible content.

## Accessibility

- Semantic HTML is used.
- Buttons have accessible names.
- Keyboard focus is visible.
- Color is not the only state indicator.
- Motion respects reduced-motion preferences.

## Mobile

- Works at 360px width.
- Main question text fits comfortably.
- Tap targets are easy to hit.
- No hover-only behavior.
